# 아키텍처 개요

## 시스템 구조

```
┌──────────────────────────────────────────────┐
│                  사용자 브라우저               │
├──────────────────────────────────────────────┤
│  Next.js Static Export (GitHub Pages)        │
│  ┌─────────┐  ┌──────────┐  ┌────────────┐  │
│  │ page.tsx │──│ Zustand  │──│ Components │  │
│  │ (Entry)  │  │ Store    │  │ (7 React)  │  │
│  └─────────┘  └──────────┘  └────────────┘  │
│       │              │              │        │
│  ┌─────────┐  ┌──────────┐  ┌────────────┐  │
│  │ Hooks   │  │ Leaflet  │  │ Filters    │  │
│  │ (Filter)│  │ Map      │  │ (Panel,    │  │
│  └─────────┘  └──────────┘  │  Search)   │  │
│                              └────────────┘  │
├──────────────────────────────────────────────┤
│  public/data/*.json (사전 빌드된 정적 데이터)  │
└──────────────────────────────────────────────┘

┌──────────────────────────────────────────────┐
│           데이터 빌드 파이프라인 (오프라인)     │
│  Excel/Web → build scripts → VWorld API      │
│           → Geocoding → JSON files           │
└──────────────────────────────────────────────┘
```

## 디자인 패턴

- **Static Site Generation**: 서버 없이 정적 파일만으로 서비스
- **Client-Side State**: Zustand로 전체 앱 상태를 클라이언트에서 관리
- **Marker Clustering**: leaflet.markercluster로 대량 마커 성능 최적화
- **Region Partitioning**: 지역별 데이터 파일 분리로 초기 로딩 시간 단축
- **Progressive Web App**: 오프라인/모바일 홈 화면 설치 지원

## 시스템 경계

- **런타임 외부 의존성**: OpenStreetMap 타일 서버만 사용 (API 키 불필요)
- **빌드 타임 외부 의존성**: VWorld API (지오코딩), localdata.go.kr (모범식당 데이터)
- **데이터 저장**: localStorage (즐겨찾기), 정적 JSON 파일 (식당 데이터)
